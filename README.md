## Как запустить?

### Для прода (ну или просто проверить, что у вас запускается все):

```bash
docker-compose up -d
```

### Для разработки:

1. Проще будет запустить контейнер как для прода, чтобы создалась бд со всеми приколами и не париться. Но если прям сильно хочется, то можно сделать отдельный компоуз для разработки, но как будто это не особо нужно

2. Если запустили контейнер, то лишнее лучше выключить, оставить только бд.

3. Потом идем в backend, устанавливаем библиотеки, применяем миграции, желательно создать суперпользователя и запускаем django:

```bash
cd backend
# Установка зависимостей
pip install -r requirements.txt

# Применение миграций
python manage.py migrate

# Создание суперпользователя (опционально)
python manage.py createsuperuser

# Запуск сервера
python manage.py runserver
```

4. Потом идем в frontend, устанавливаем все зависимости и запускаем:

```bash
cd frontend
npm install
npm run dev
```

## Важно

У вас скорее всего pycharm не найдет venv, поэтому его либо ручками включаем, либо можно открыть pycharm'ом папку не 4tutor, а сразу backend, но в таком случае, не получится иметь доступ к файлам выше папки backend.
